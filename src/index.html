Compilação pelo Grunt

Este projeto serve para mostrar o efeito de compilação que o Grunt pode gerar, automatizando o processo de desenvolvimento e permitindo a divisão entre os ambientes de desenvolvimento e produção.

HTML

Para o HTML utilizamos tanto o htmlmin para minimizar o tamanho do final do arquivo e portanto ganhar performance no carregamento quanto o replace para modificar os arquivos para o qual o HTML aponta no ambiente de produção.

LESS

O CSS deste projeto foi feito utilizando o LESS, e depois compilado pelo Grunt para o CSS em uma versão de produção que será exibida ao publicar o site. Isto permite a separação dos ambientes e também permite que outros desenvolvedores consigam instalar as dependências e trabalhar no projeto de maneira fácil, enquanto também entrega apenas a informação necessária ao usuário final, permitindo mais agilidade no carregamento devido ao tamanho reduzido, assim como também retirando possivel informação irrelevante ao usuário final.

javaScript

Baseado no projeto deste módulo o formulário abaixo gera um número aleatório dentro do parâmetro determinado, entretanto ele exibe o número arredondado utilizando as três variações diferentes do “Math” do javaScript. Assim podemos ver as possíveis variações que a linguagem nos permite O código foi compilado pelo Grunt para a versão de distribuição.

É importante destacar que o arredondamento não possui ajustes(+1 por exemplo) e logo serve para mostrar as diferenças no arredondamento mesmo quando as mesmas fogem dos parâmetros do input.

Também utilizamos o uglify para minimizar o tamanho do arquivo final assim ganhando performance no carregamento para o usuário final no ambiente de produção.
